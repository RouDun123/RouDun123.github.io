SQL通用语法

1、SQL语句可以单行或多行书写，以分号结尾。

2、SQL语句可以使用空格/缩进来增强语句的可读性。

3、MySQL数据库的SQL语句不区分大小写，关键字建议使用大写。

4、注释：

  • 单行注释：– 注释内容 或者 # 注释内容（MySQL 特有）
  • 多行注释：/* 注释内容 */

语言分类

分类 全称 说明
DDL Date Definition Language 数据定义语言，用来定义数据库对象（数据库、表、字段）
DML Date Manipulation Language 数据操作语言，用来对数据库表中的数据进行增删改
DQL Date Query Language 数据查询语言，用来查询数据库中的表的记录
DCL Date Control Language 数据控制语言，用来创建数据库用户、控制数据库的访问权限

==创建库==

右键 MySQL-@localhost —— New —— Schema 点击—— 给数据库命名 —— 点击 Ok ;

==创建表==

右键 某库 —— New —— Table 点击 —— 给表命名 —— comment 给表添加注释信息 —— 点击 左上角 “+” —— columns —— 添加字段 ——设置字段名称、注释、数据类型 —— 多次点击columns ，添加多个字段。—— 字段添加完成后 —— 点击 Ok;

==给表添加字段==

右键 某表 —— modify Table —— 点击 左上角 “+” —— columns —— 添加字段 ——设置字段名称、注释、数据类型 —— 多次点击columns ，添加多个字段。—— 字段添加完成后 —— 点击 Ok;

==切换成命令行形式==

右键 某数据库 —— new —— Query console —— 输入命令行 —— 选中执行；

数据类型

MySQL中的数据类型有很多，主要分为三类：数值类型、字符串类型、日期时间类型。

==数值类型==

类型 大小 有符号（signed)范围 无符号（unsigned)范围 描述
tinyint 1 byte ( -128 , 127 ) ( 0 , 255 ) 小整数值
smallint 2 bytes ( -32768 , 32767 ) ( 0 ,65535 ) 大整数值
mediumint 3 bytes ( -8388608 , 8388607 ) ( 0 , 16777215 ) 大整数值
int / integer 4 bytes ( -2147483648 , 2147483647 ) ( 0 , 4294967295 ) 大整数值
bigint 8 bytes ( -2^63 , 2^63 - 1 ) ( 0 , 2^64 - 1 ) 极大整数值
float 4 bytes ( -3.402823466 E+38 , 3.402823466351 E+38 ) 0 和 ( 1.175494351 E-38 , 3.402823466 E+38 ) 单精度浮点数值
double 8 bytes ( -1.7976931348623157 E+308 , 1.7976931348623157 E+308 ) 0 和 ( 2.2250738585072014 E-308 , 1.7976931348623157 E+308 ) 双精度浮点数值
decimal 依赖于M（精度）和 D（标度）的值 依赖于M (精度) 和 D （标度）的值 小数值（精确定点数）

注意：

  • 精度M，指的是数字的位数。

    标度D，指的是数字的小数位数。

​ 例如：123.45；

​ 精度M ：5

​ 标度D ：2

  • 举例：

    年龄 ：age tinyint unsigned //表示 年龄是无符号整型；

    分数： score double(4 , 1) // 4:表示 精度M , 1:表示标度D

==字符串类型==

类型 大小 描述
char 0 - 255 bytes 定长字符串
varchar 0 - 65535 bytes 变长字符串
tinyblob 0 - 255 bytes 不超过255个字符的二进制数据
tinytext 0 - 255 bytes 短文本字符串
blob o - 65535 bytes 二进制形式的长文本数据
text 0 - 65535 bytes 长文本数据
mediumblob 0 - 16777215 bytes 二进制形式的中等长度文本数据
mediumtext 0 - 16777215 bytes 中等长度文本数据
longblob 0 - 4294967295 bytes 二进制形式的极大文本数据
longtext 0 - 4294967295 bytes 极大文本数据

注意：

  • 当我们使用 char 或者 varchar 时，后面都跟一个参数，这表示，字符串的最大长度。

  • char(10) ：即使使用1个字符，也占用10个字符的空间，其他9个字符的空间，会用 空格 进行 补位；

  • varchar(10) ：在最大存储字符个数内，存储多少个字符，就占多少空间；

  • char 与 varchar 相比较，char 的 性能较好；

  • 举例：

    用户名 username varchar(50)

    性别 gender char(1)

设计一张员工信息表，要求如下：

7、入职时间（取值年月日即可 ）
